# Contributing to ANIA Player

Thank you for your interest in ANIA Player!

## What We Accept

- Bug reports
- Feature suggestions
- Documentation improvements
- Translation error reports (194 languages — quality corrections are welcome)

## What We Don't Accept

- Code contributions (source code is proprietary)
- Pull requests modifying core functionality
- Requests to open-source the code

---

## How to Contribute

### Bug Reports

1. Check [existing issues](../../issues) to avoid duplicates
2. Open a new issue and include:
   - A clear description of the problem
   - Steps to reproduce
   - Expected vs. actual behavior
   - OS, app version, and platform (Windows / Linux x86_64 / Linux ARM64)
   - Screenshots or logs if applicable

### Feature Requests

1. Search existing issues first
2. Describe the feature and the use case it solves
3. Explain how it benefits users

### Security Issues

**Do not** open public issues for security vulnerabilities.

Email: support@aniamodels.shop — See [SECURITY.md](SECURITY.md) for details.

---

## Other Ways to Contribute

- **Create avatars** using Ania Creators and publish them on [aniamodels.shop](https://aniamodels.shop) — growing the library benefits everyone
- **Accept commissions** — if you're a skilled creator, other users and companies may contact you to build custom avatars
- **Leave feedback** on the marketplace — reviews help creators improve their work
- **Share the project** with streamers, developers, and content creators who could benefit from it

---

## Code of Conduct

Be respectful, constructive, and professional in all interactions.
